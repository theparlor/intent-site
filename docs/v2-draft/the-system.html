▸ Zone 2 of 4 · The System

How the loop actually runs

The System zone covers the methodology layer: the Notice → Spec → Execute → Observe loop, the work ontology it produces, the spec-shaping protocol that turns intents into agent-ready specs, and the dogfood evidence from Intent building itself. It's less technical than The Build and more operational than The Hypothesis.

The four-phase continuous loop. No sprint boundaries. Multi-speed (some loops run in minutes, some in weeks). Every transition emits an event.

▸ The four phases (with ancestors)

▲ Notice

What's the signal?

Capture friction, questions, observations, anomalies from anywhere work happens. Every signal is atomic and scored on trust dimensions.

Ancestors: Continuous Discovery (Torres), OODA Observe (Boyd), Get out of the building (Blank)

◆ Spec

What would we build?

Shape signals into agent-ready specifications with verifiable contracts. The four-persona interrogation protocol runs here.

Ancestors: Story Mapping (Patton), TDD (Beck), Shape Up (Singer), Outcomes (Seiden)

● Execute

What actually gets done?

Agents implement against contracts. Trust-gated at L0–L4 autonomy levels. Contract assertions pass or fail with evidence.

Ancestors: Continuous Delivery (Humble/Farley), DORA (Forsgren), GIST (Gilad)

○ Observe

What did we learn?

Runtime feedback turns into new signals. Single-loop: spec drift. Double-loop: questioning assumptions (still under construction).

Ancestors: PDCA (Deming), Build-Measure-Learn (Ries), Double-loop (Argyris), Three Ways (Kim)

▸ Open gap: double-loop learning

The Observe phase claims to implement Argyris's double-loop learning (questioning assumptions, not just optimizing execution). The current mechanism is single-loop only — we detect spec drift, we don't question whether the goals were right. The 178-persona library is the latent mechanism for double-loop, framed as execution help instead of premise challenge. Work to convert this is tracked as SIG-050 / INT-013. This is an honest gap, not a capability.
